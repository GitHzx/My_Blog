LEAStereo 和 NAS

2021-03-06 34 min read

前言

  本文章《Hierarchical Neural Architecture Search for Deep Stereo Matching》是第一个在双目立体视觉任务上同时引入cell-level和network-level神经网络结构搜索(NAS)的工作。该工作于2020年由Xuelian Cheng，Yiran Zhong，Mehrtash，Harandi，Yuchao Dai，Xiaojun Chang，Tom Drummomd，Hongdong Li，Zongyuan Ge提出，来自于澳大利亚多个大学及研究机构。

  文章提出的LEAStereo模型应用了2019年的DARTS在cell-level上搜索重复使用的单元结构，然后应用了2019年的Auto-Deeplad来搜索最优的网络结构，整个网络由相同的单元连接而成。LEAStereo模型以最小的参数量、很快的运行速度在多个数据集SceneFlow、KITTI2012、KITTI2015、MiddleBury上取得了最优的表现，并且在KITTI数据集上一直保持SOTA表现至今。

摘要

  尽管NAS已经成功用于解决多种高级视觉任务，但在低级视觉任务上，如立体匹配，其表现并不尽如人意。这某种程度上是因为深度立体匹配网络模型通常被人为设计得很大，为NAS带来了巨大的搜索空间和时间。这篇文章构建了第一个用于深度立体匹配的端到端的分级NAS框架，该框架遵循了传统的立体匹配pipeline，同时利用了与任务密切相关的几何先验信息。整个框架的优化基于端到端的一体式监督训练。

背景

  基于深度学习的深度立体匹配方法始于MC-CNN，之后大量的深度匹配网络模型被构建出来，这些模型可以被粗略地分成两类：1.直接回归方法，2.代价体积方法。后者实际上是对传统pipeline的模拟和优化。

直接回归方法

  直接回归方法通过构建网络直接根据输入图像进行像素级的深度回归预测，完全是由数据驱动的(data-driven)，没有考虑任何的几何约束、先验信息。这种方法大都采用大型的U-shape 2D编码-解码网络来回归深度图，其完全基于数据驱动的模式带来了不可忽视的模型泛化性问题，例如，DispNet无法通过随机的立体匹配测试。

代价体积方法

  实际上，体积代价方法遵循了半全局立体匹配的基本思想：首先构建一个4D的特征矩阵，包括图像长、图像宽、视差范围、特征四个维度；然后对4D特征矩阵进行卷积处理，以计算正则化后的3D代价矩阵，包括图像长、图像宽、视差范围；最后选取最小的代价对应的视差作为预测的最优视差值。

  体积代价方法通常构建两个网络模块：1.feature net，2.matching net。前者通过2D卷积层从输入图像对中提取特征矩阵，后者利用3D卷积核从4D特征矩阵中计算匹配代价。目前，体积代价方法在各种数据集上取得了最好的表现，不同的模型设计了不同的feature net和matching net，因而这类方法的核心问题就是给feature net和matching net设计好的网络结构。此处说明了整篇文章最大的Motivation。

NAS

  Neural Architecture Search在多个高级视觉任务中取得了成功，如分类、目标检测、语义分割。然而，直接应用NAS去寻找立体匹配网络的困难在于：构建代价矩阵本身就需要耗费大量内存，甚至某些方法训练单个patch就要耗费6 8个Gigabytes的GPU内存，因此，通过数据训练从大量可能的网络架构中搜索最优架构更是耗费巨大。

  这篇文章参考NAS相关工作构建了一种可以在给定网络架构空间中自动搜索最优feature net和matching net网络结构的NAS pipeline。该算法不仅仅可以搜索网络的结构，还可以搜索特征图的尺寸、特征矩阵的体积、输出深度图的尺寸。现代CNN网络模型设计通常遵循两级分层结构(two-level hierarchy)，其基本架构是由多个重复性的单元网络模块连接成的整体网络，其中内部单元级网络控制特定的分层计算，外部网络级控制空间分辨率变化。cell-level的NAS的目的就是寻找结构最优的内部单元网络模块，network-level的NAS的目的则是寻找多个单元网络模块的整体最优连接网络。与AutoDispNet只能搜索cell-level最优结构不同的是，该文章的LEAStereo还可以搜索network-level的最优结构。

方法

方法思想

  该文章首先构建针对该任务的网络结构搜索空间。研究者首先回顾了以往的模型，将两个网络的基本结构确定为最常用的编码-解码器；然后效仿面向语义分割的Auto-DeepLab提出了两级分层搜索策略，从而对每个网络都能在最小组成单元上和整体网络上分别搜索最优结构；鉴于纯数据驱动的U-shape通常是难以训练的，基于代价体积的方法则通过引入几何先验实现了更快的收敛和更好的性能。

  针对立体匹配网络巨大的尺寸这一问题，该文章引入几何先验将整个网络及数据流分解成多个串联模块，实际上是模拟了传统的pipeline：1.2D feature net，2.4D feature volume，3.3D matching net，4.3D cost volume，5.soft-argmin layer。其中，只有2D feature net和3D matching net的需要进行训练，NAS正是用于两个网络的搜索。

DARTS与cell-level Search

DARTS简介

  本部分是方法实现的第一个关键点，此处主要参考文章《Darts: Differentiable architecture search.》。以往的NAS是在离散空间上进行黑盒优化，利用各种搜索优化策略，如强化学习、遗传进化学习、贝叶斯优化等方法寻找特定的结构并进行评估，这些方法耗时严重。DARTS的突出贡献是将离散的搜索空间"弱化"为连续的搜索空间，然后提供了一套双层bilevel优化策略直接应用随机梯度下降法进行优化搜索。DARTS不仅仅可以搜索特定的结构，如卷积核尺寸，更可以搜索完整的单元网络结构，即，"一种具有复杂图形拓扑结构的高性能网络框架"。

DARTS算法说明

  单元级网络模块Cell实际上是一个小的全卷积模块，它通常会在整个神经网络中重复使用很多次，DARTS就是用于搜索一个具有最优结构的Cell。Cell被定义为一种具有N\mathcal{N}N个节点Node的全连接有向无环图(DAG)，其中每个Node代表一组feature map，Node之间的有向边代表一类由前向后的操作。Cell既可以不断地堆叠组成CNN，也可以循环连接组成RNN。

cell-level搜索空间

  本任务则寻求构建一种CNN的Cell，该Cell的基本结构具有两个输入Nodes，一个输出Node，三个中间Nodes。第lll层的输出Node记为Cl\mathcal{C}_{l}Cl​，而第lll层的输入Node则为第l−2l-2l−2层与第l−1l-1l−1层的输出Nodes即Cl−2\mathcal{C}_{l-2}Cl−2​、Cl−1\mathcal{C}_{l-1}Cl−1​；所有可能的由前向后的操作组成的操作集记为O\mathcal{O}O，这里规定操作集只包含不同尺寸的convolution、skip connection。内部第jjj个Node记为S(j)\mathcal{S}^{(j)}S(j)，其定义为：

S(j)=∑i→j=o(i,j)(S(i))\mathcal{S}^{(j)} = \sum_{i\to j} = o^{(i,j)}(\mathcal{S}^{(i)}) S(j)=i→j∑​=o(i,j)(S(i))

其中，o(i,j)o^{(i,j)}o(i,j)表示由S(i)\mathcal{S}^{(i)}S(i)到S(j)\mathcal{S}^{(j)}S(j)的操作。

  cell-level网络结构搜索的目的是确定全连接DAG上所有有向边所代表的最优操作类型，找出具有最优结构的Cell。

Bilevel优化策略

  鉴于操作连续化方法引入了新的结构权重向量α\alphaα，DARTS使用了双层优化策略来实现对结构参数α\alphaα以及卷积核参数www的梯度下降优化。具体来说，DARTS既需要找到一个最优的w∗w^{*}w∗向量来学习每个Cell内的卷积核参数，还需要找到一个最优的α∗\alpha^{*}α∗向量来学习每个Cell内feature tensor之间的操作类型。由于存在两个参数向量，DARTS采用了一种近似的Bilevel优化策略来同时对两个参数向量进行梯度下降，既保证了优化效果，又降低了优化时间。

  在实际的学习过程中，DARTS将数据集分为训练集和验证集并分别在其上定义损失Ltrain\mathcal{L}_{train}Ltrain​和Lval\mathcal{L}_{val}Lval​，训练集损失用来优化卷积核参数www，验证集损失用来优化结构参数α\alphaα。由此可以定义一个双层优化问题，其中www是lower-level的变量，α\alphaα是upper-level的变量，双层策略即是先优化lower-level的变量www，然后在w∗w^{*}w∗的基础上对upper-level的变量α\alphaα进行优化：

Auto-DeepLab与network-level Search

Auto-DeepLab简介

  本部分是方法实现的第二个关键点，此处主要参考文章《Auto-DeepLab: Hierarchical Neural Architecture Search for Semantic Image Segmentation.》 。

  以往的NAS存在的第二个问题是无法进行网络级的结构搜索，它们一旦找到最优的单元结构就使用预定义的网络结构将多个Cell进行连接。Auto-DeepLab的核心贡献就是提出了网络级的结构搜索方法，它一方面应用DARTS构建单元级的网络搜索空间，另一方面则通过首创的方法构建网络级的结构搜索空间；随后提供了一整套的优化策略应用随机梯度下降法进行优化搜索，并在优化收敛之后通过Viterbi算法解码最优网络架构。Auto-DeepLab是首个在密集图像预测任务上应用NAS的方法，尽管它最初被用来解决图像语义分割问题，但同样在立体匹配上取得了良好的效果。

Auto-DeepLab中的network-level算法说明

  我们主要关注Auto-DeepLab中原创性的工作，即network-level网络结构搜索空间的构建方法与整个网络的优化搜索及解码策略。

  Auto-DeepLab中network-level网络结构搜索空间被定义为一种具有多个节点Node的有向无环图(DAG)，其中每个Node代表一个单元模块Cell，Node之间的有向边则代表一类由前向后的网络级操作，用于实现不同空间分辨率的变换。这里注意与Cell搜索空间中的节点和有向边的区别。

network-level搜索空间

  Auto-DeepLab的研究者们在用于密集图像预测的诸多网络架构中发现了一种共通的模式：这些而网络架构通常由"Normal Cells"和"Reduction Cells"共同组成，其中"Normal Cells"保持特征张量的空间尺寸与维数，"Reduction Cells"则在将特征张量的尺寸缩小一倍的同时将其维数扩充一倍。精简地说，这里存在着两个默认的规则：

  • 下一层feature-tensor的空间尺寸是上一层两倍大、或两倍小、或保持不变。

  • 最大的空间尺寸是原图的分辨率，最小的空间尺寸则为下采样32倍。

  Auto-DeepLab遵循以上的两个规则，其network-level的网络结构搜索空间定义为：网络起始端为两个stem"枝干"结构，即灰色部分，每个枝干都将feature tensor的空间尺寸降低2倍；之后，水平方向上共有lll列Cell Nodes，同一列的Cell Nodes由上向下空间分辨率依次缩小一倍；竖直方向上共有4行Cell Nodes，同一行的CellN odes的空间分辨率相同，最上面一行空间分辨率为输入图像的14\frac{1}{4}41​，最下面一行空间分辨率为输入图像的132\frac{1}{32}321​，即蓝色部分。

  注意到，每一个蓝色Cell Node，也包括第三个灰色Node，都通过两个或三个有向边连接后一个Cell Node。这实际上对应了上述第一条规则：下一层feature-tensor的空间尺寸是上一层两倍大、或两倍小、或保持不变。位于顶层的Cell Node只能进行空间尺寸的缩小和保持操作，而底层则只能进行放大和保持操作，故二者存在两条有向边。目前该空间还未包括U-net架构。

  network-level的网络结构搜索的目的是在整个DAG网络上，即所有的蓝色Cell Nodes及其相连的有向边之上，搜索一条最优的空间尺寸变换路径。

一阶近似Bilevel优化策略

  Auto-DeepLab既需要找到一个最优的w∗w^{*}w∗向量来学习每个Cell内的卷积核参数，还需要找到一个最优的α∗\alpha^{*}α∗向量来学习每个Cell内feature tensor之间的操作类型，同时需要找到一个最优的β∗\beta^{*}β∗来学习多个Cell连接而成的网络结构。

  Auto-DeepLab采用了与DARTS的Bilevel优化思想，卷积核参数www是lower-level变量，结构参数α,β\alpha,\betaα,β是upper-level变量。与之不同的是，Auto-DeepLab分别独立地优化卷积核参数www和结构参数α,β\alpha,\betaα,β，相当于使用Bilevel的一阶近似。Auto-DeepLab将数据集分为两个不相交的训练集trainA和trainB，Bilevel优化在两个训练集上交替进行：

  • 通过损失梯度∇wLtrainA(w,α,β)\nabla_{w} \mathcal{L}_{trainA}(w,\alpha,\beta)∇w​LtrainA​(w,α,β)更新卷积核参数www。

  • 通过损失梯度∇α,βLtrainB(w,α,β)\nabla_{\alpha,\beta} \mathcal{L}_{trainB}(w,\alpha,\beta)∇α,β​LtrainB​(w,α,β)更新结构参数α,β\alpha,\betaα,β。

解码离散架构

  与DARTS一样，单元级网络依然采用argmax函数来选择最可能的操作。

  Auto-DeepLab的研究者将β\betaβ值解释为在不同"时间步长"(列数)的不同"状态"(空间分辨率)之间的"迁移概率"，优化目标是从头到尾找到"最大概率"的路径。因此Auto-DeepLab使用经典的Viterbi算法解码该最优路径。

LEAStereo的实现细节

  LEAStereo在整体框架上如前所述，使用DARTS搜索一种空间分辨率固定的Cell，使用Auto-DeepLad为不同Cell选择合适的空间分辨率并进行连接。LEAStereo会预先检查邻近连接的Cell的空间分辨率，并采用upsampling\\verb|\|\downsampling把不同的空间分辨率变换至相同。需要注意的是，LEAStereo在具体实现上对DARTS和Auto-DeepLab中的方法进行了微调，使之在立体匹配任务上表现得更好。

cell-level Search的实现细节

  LEAStereo对DARTS的改动已经在之前部分的被展现出来，如，单元结构Cell的输入Nodes、隐藏Nodes、输出Node分别被定义为2个、3个、1个；操作集O\mathcal{O}O只包含convolution与skip connection操作，而不包括dilated convolution、max\\verb|\|\averagepo oling、identity、zero操作。

  更进一步地，之前的Cell的输出Noder都是简单地把隐藏Nodes的feature tensor拼接(concatenate)起来，作者则借鉴了残差网络，使用residual connection将输入NodeCl−1\mathcal{C}_{l-1}Cl−1​的连接到输出Node上，如图3中红色虚线所示。

  除此之外，作者观察到移除操作集中的dilated convolution及pooling不会影响到网络性能。因而，feature net的操作集OF\mathcal{O}^{F}OF与matching net的操作集OM\mathcal{O}^{M}OM被更具体地设计为：

  • OF={3×3  2D convolution,  skip connection}\mathcal{O}^{F}=\{ 3\times3 \ \ 2D \ convolution,\ \ skip \ connection \}OF={3×3  2D convolution,  skip connection}

  • OM={3×3×3  3D convolution,  skip connection}\mathcal{O}^{M}=\{ 3\times3\times3 \ \ 3D \ convolution,\ \ skip \ connection \}OM={3×3×3  3D convolution,  skip connection}

network-level Search的实现细节

  LEAStereo对Auto-DeepLad的改动主要是体现在两个超参数：1.最低的空间分辨率。2.搜索空间层(列)数L。

  研究人员经验主义地将最低分辨率设置为124\frac{1}{24}241​，同时将空间分辨率由高到低设置为4类，其下采样率为{3,2,2,2}\{3,2,2,2\}{3,2,2,2}。使用{3,2,2,2}\{3,2,2,2\}{3,2,2,2}的下采样率而非{2,2,2,2,2}\{2,2,2,2,2\}{2,2,2,2,2}带来的好处不用进行两次下采样，他们同样经验主义地发现这几乎不会影响网络的表现。在feature net起始端，研究人员使用三个卷积层组成枝干(stem)结构来实现13\frac{1}{3}31​的下采样：3×33\times33×3 convolution with 3 strides，3×33\times33×3 convolution with 1 stride，3×33\times33×3 convolution with 1 stride。

  对于搜索空间层(列)数L，研究人员经验主义地观察到将feature net的层数LFL^{F}LF与matching net的层数LML^{M}LM分别设置为6和12能够在计算速度和性能上实现良好的平衡，这远小于人工设计的网络层数。

  鉴于LEAStereo端到端的结构，LEAStereo在输出的视差图上进行监督训练，从而使feature net和matching net实现联合学习。

实验结果

  研究人员采用SceneFlow数据集作为训练数据集来搜索最优的架构。然后通过KITTI 2012，KITTI 2015和Middlebury 2014 Benchmark对架构进行评估。在消融实验研究中，研究人员分析了不同的搜索空间以及不同搜索策略对LEAStereo的影响。

网络搜索实验

  SceneFlow数据集包含35454个训练和4370个测试合成图像，典型图像尺寸为540×960。研究人员使用"finalpass"版本，从训练集中随机选择20000个图像对作为搜索训练集，然后从训练集中另选1000个图像对作为搜索验证集，二者不相交。LEAStereo网络在Pytorch上实现，使用了大小为192×384的随机裁剪进行数据增强。

总结

  LEAStereo并无理论上的创新，依然延续了由传统pipeline演变而来的深度立体匹配网络的基本结构，loss函数的设计也没有引入新的几何约束。其最大的贡献就是通过对NAS很好的应用而大大降低了深度立体匹配网络的参数量，同时保证甚至加强了模型的精度表现。

  由此可以看到，CNN的学习能力是强大的，好的模型应遵循三个关键点：数据、先验约束、优化策略。研究人员特别地指出代价体积方法因遵循了几何约束而降低了计算耗费、加强了表现以及泛化性能。由此，可以做一个猜测：在高质量的数据集上施加适当而合理的先验约束，通过高效的优化算法就可以自发地学习一个SOTA的网络。除了网络参数，我们甚至无需关注大部分结构，既然CNN在很大程度上是不可解释的，不妨把大部分的网络结构也交给优化方法。
